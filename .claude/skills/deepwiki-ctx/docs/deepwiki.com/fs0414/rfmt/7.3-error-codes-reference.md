This document provides a comprehensive reference for all error codes, exit codes, and error handling mechanisms in rfmt. It covers the error reporting system used by the CLI, Ruby API, and Rust core, along with troubleshooting guidance for common issues.

For general usage information, see [User Guides](https://deepwiki.com/fs0414/rfmt/3-user-guides.html). ## Overview

rfmt implements a structured error reporting system with standardized error codes, detailed error messages, and actionable troubleshooting guidance. Errors can originate from multiple layers:

* **Ruby Layer**: Configuration loading, file I/O, CLI operations
* **FFI Boundary**: Data serialization, type conversion
* **Rust Layer**: AST parsing via Prism, formatting logic, internal processing

All errors include an error code (E001-E999) that categorizes the issue and provides a reference point for troubleshooting.

## CLI Exit Codes

The `rfmt` CLI uses standard UNIX exit codes to indicate the result of operations. These exit codes are particularly important for CI/CD integrations and scripting.

| Exit Code | Meaning | Usage Context |
| --- | --- | --- |
| `0` | Success | All files formatted successfully, or no changes needed |
| `1` | Error | Parse errors, I/O errors, configuration errors, or formatting failures occurred |
| `2` | Formatting Required | Files need formatting (only returned when using `--check` mode) |
| `130` | Interrupted | Process was interrupted by user (Ctrl+C) |

## Error Code Categories

rfmt error codes follow a structured numbering scheme:

* **E001-E099**: Parser and syntax errors
* **E100-E199**: Configuration errors
* **E200-E299**: I/O and file system errors
* **E300-E499**: Formatting and transformation errors
* **E500-E899**: Integration and external errors
* **E900-E999**: Internal errors and bugs

## Error Message Format

All rfmt errors follow a standardized format for consistency and clarity:

```
[Rfmt::ErrorType] Error in file.rb:line:column
Error description with context

Code:
   line-2 | code before
   line-1 | code before
   line   | problematic code
          | ^^^^^^^
   line+1 | code after

Help: https://rfmt.dev/errors/EXXX
```

### Error Message Components

| Component | Description | Example |
| --- | --- | --- |
| Error Type | Ruby exception class | `Rfmt::ParseError` |
| Location | File path, line, and column | `example.rb:5:10` |
| Description | Human-readable explanation | "Expected closing 'end' for class definition" |
| Code Snippet | Context with error indicator | Shows 2 lines before/after with caret marker |
| Help URL | Link to detailed documentation | `https://rfmt.dev/errors/E001` |

**Sources:** [docs/user\_guide.md327-341](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.md#L327-L341) [docs/user\_guide.ja.md329-342](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.ja.md#L329-L342)

## Common Error Scenarios

### E001: Parse Error (Syntax Error)

**Error Type:** `Rfmt::ParseError`
**Severity:** Critical
**Layer:** Ruby/Prism Parser

#### Causes

* Missing `end` keyword for `class`, `module`, `def`, `if`, `case`, etc.
* Unclosed string literals or interpolations
* Invalid Ruby syntax
* Mismatched brackets, parentheses, or braces

#### Example

#### Error Message

```
[Rfmt::ParseError] Parse error in user.rb:3:3
Expected closing 'end' for method definition

Code:
   1 | class User
   2 |   def initialize
   3 |     @name = name
     |     ^
   4 | end

Help: https://rfmt.dev/errors/E001
```

#### Resolution

Fix the syntax error before running rfmt. Use `ruby -c file.rb` to check syntax independently.

**Sources:** [docs/user\_guide.md343-366](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.md#L343-L366) [spec/rfmt\_spec.rb24-30](https://github.com/fs0414/rfmt/blob/90c575e6/spec/rfmt_spec.rb#L24-L30)

---

### E002: Configuration Error

**Error Type:** `Rfmt::ConfigError` / `Rfmt::Error`
**Severity:** High
**Layer:** Ruby Configuration Loading

#### Causes

* Invalid YAML syntax in `.rfmt.yml`
* Wrong data types for configuration values
* Out-of-range values (e.g., `line_length: 10`)
* Missing required configuration keys
* Configuration file not found (when explicitly specified)

#### Example

#### Error Message

```
[Rfmt::Error] Invalid YAML in configuration file .rfmt.yml
Configuration file syntax error at line 3

Help: https://rfmt.dev/errors/E002
```

#### Resolution

Validate your `.rfmt.yml` against the schema. Key validation rules:

* `line_length`: Integer between 40-500
* `indent_width`: Integer between 1-8
* `indent_style`: String, either "spaces" or "tabs"
* `quote_style`: String, either "double" or "single"

**Sources:** [lib/rfmt/configuration.rb71-77](https://github.com/fs0414/rfmt/blob/90c575e6/lib/rfmt/configuration.rb#L71-L77) [spec/config\_management\_spec.rb82-91](https://github.com/fs0414/rfmt/blob/90c575e6/spec/config_management_spec.rb#L82-L91) [docs/user\_guide.md367-382](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.md#L367-L382)

### E003: I/O Error

**Error Type:** `Rfmt::IoError` / `Rfmt::Error`
**Severity:** High
**Layer:** Ruby File System Operations

#### Causes

* File not found
* Permission denied (read or write)
* Disk full
* File locked by another process
* Directory not accessible

#### Resolution

* Verify file paths and permissions
* Check disk space with `df -h`
* Ensure files are not open in other applications
* Check file ownership with `ls -la`

**Sources:** [lib/rfmt/cli.rb172](https://github.com/fs0414/rfmt/blob/90c575e6/lib/rfmt/cli.rb#L172-L172) [lib/rfmt/cli.rb215](https://github.com/fs0414/rfmt/blob/90c575e6/lib/rfmt/cli.rb#L215-L215)

### E006: Unsupported Feature

**Error Type:** `Rfmt::UnsupportedFeature`
**Severity:** Medium
**Layer:** Rust Formatting Engine

#### Causes

* Ruby syntax that rfmt doesn't fully support yet
* Advanced metaprogramming constructs
* Experimental Ruby features
* Edge cases in AST reconstruction

#### Example

#### Resolution

1. Check the [ROADMAP](https://github.com/fs0414/rfmt/blob/90c575e6/ROADMAP) for planned support
2. File an issue with a minimal reproduction case
3. Use `--check` mode to identify unsupported files without modifying them
4. Add problematic files to `exclude` patterns temporarily

**Sources:** [docs/user\_guide.md383-388](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.md#L383-L388)

---

### E999: Internal Error

**Error Type:** `Rfmt::InternalError`
**Severity:** Critical
**Layer:** Any (indicates a bug)

#### Causes

* Unexpected internal state
* Assertion failure in Rust code
* Unhandled edge case
* Memory corruption (rare)

#### Resolution

This is a bug in rfmt. Please report it with:

1. Full error message and stack trace
2. Minimal reproduction case (Ruby code that triggers the error)
3. rfmt version (`rfmt version`)
4. Ruby version (`ruby --version`)
5. Operating system and architecture

Enable debug logging for more details:

**Sources:** [docs/user\_guide.md429-456](https://github.com/fs0414/rfmt/blob/90c575e6/docs/user_guide.md#L429-L456)

## Configuration Validation Errors

### Validation Rules

The configuration system validates all settings before applying them:

| Setting | Validation Rule | Error on Violation |
| --- | --- | --- |
| `formatting.line_length` | Must be integer > 0, typically 40-500 | E002: ConfigError |
| `formatting.indent_width` | Must be integer > 0, typically 1-8 | E002: ConfigError |
| `formatting.indent_style` | Must be "spaces" or "tabs" | E002: ConfigError |
| `formatting.quote_style` | Must be "double" or "single" | E002: ConfigError |
| `include` | Must be array of strings | E002: ConfigError |
| `exclude` | Must be array of strings | E002: ConfigError |

## Error Prevention Best Practices

1. **Run syntax check first**: Use `ruby -c` before running rfmt
2. **Version control configuration**: Commit `.rfmt.yml` to repository
3. **Pin rfmt version**: Use `~> X.Y.Z` in Gemfile for stability
4. **Test incrementally**: Format files in batches, not all at once
5. **Use cache**: Enable caching with `--cache` to skip validated files
6. **Enable verbose mode during debugging**: Use `--verbose` for troubleshooting
7. **Review diffs**: Use `--diff` to preview changes before writing
8. **Handle errors gracefully**: Wrap API calls in `begin/rescue` blocks

## Error Reporting

If you encounter an error that you believe is a bug in rfmt:

1. **Check existing issues**: Search [GitHub Issues](https://github.com/fs0414/rfmt/blob/90c575e6/GitHub Issues)
2. **Gather information**:
   * Full error message and code
   * Minimal reproduction case
   * rfmt version (`rfmt version`)
   * Ruby version (`ruby --version`)
   * Operating system and architecture
3. **Enable debug logging**: Run with `RFMT_DEBUG=1 rfmt file.rb --verbose`
4. **File an issue**: Include all gathered information
